# 키 (Key)

테이블에서 각 레코드를 고유하게 식별하거나, 테이블 간의 관계를 정의하기 위한 속성

키는 데이터의 중복을 방지하고, 검색 속도를 향상 시키며, 데이터의 무결성을 보장하는 데 매우 중요한 역할을 함.

<br>

| **키(Key) 종류** | **핵심 정의** | **유일성 (Uniqueness)** | **최소성 (Minimality)** |
| --- | --- | --- | --- |
| **슈퍼키** (Super Key) | 레코드를 **유일하게** 식별할 수 있는 속성의 집합 | **O (필수)** | X (만족 안 해도 됨) |
| **후보키** (Candidate Key) | 슈퍼키 중 **최소성**을 만족하는 키 | **O (필수)** | **O (필수)** |
| **기본키** (Primary Key, PK) | 후보키 중에서 **대표로 선택된** 단 하나의 키 | **O (필수)** | **O (필수)** |
| **대체키** (Alternate Key) | 후보키 중 기본키가 되고 **남은** 키 | **O (필수)** | **O (필수)** |
| **외래키** (Foreign Key, FK) | 다른 테이블의 **기본키를 참조**하는 키 | X (관계 없음) | X (관계 없음) |

<br>

## 1. 기본키 (Primary Key, PK)

후보키 중에서 데이터베이스 설계자가 **대표로 선택한 단 하나의 키**

- 유일성 (Unique),  중복  불가
- **NULL 값을 가질 수 없음** (Not Null)
- 기본 키 칼럼은 레코드를 고유하게 식별 → 데이터 무결성 유지의 중요 요소

<br>

## 2. 외래키 (Foreign Key)

한 테이블의 속성이 다른 테이블의 기본키를 참조하는 키

- 외래 키 칼럼은 중복 허용
- **NULL 값을 가질 수 없음** (Not Null)
- 테이블 간의 관계를 맺고 연결하는 데 사용됨
- **참조 무결성(Referential Integrity)** 유지
외래키는 참조하는 테이블의 기본 키나 고유 키와 연결됨

<br>

## 3. 후보 키 (Candidate Key)

슈퍼키 중에서 최소한의 속성만으로 레코드를 고유하게 식별할 수 있는 키

- 유일성 & 최소성을 만족하는 키
- **NULL 값을 가질 수 없음** (Not Null)
- 기본키는 후보 키 중 하나를 선택한 것
후보키는 기본 키가 될 수 있는 모든 후보임 (여러 개 일 수 있음)

<br>

## 4. 대체 키 (Alternate Key)

후보 키 중에서 기본 키로 선택되지 않은 나머지 키

- 유일성 & 최소성을 만족하는 키
- **NULL 값을 가질 수 없음** (Not Null)
- 기본키 외에 레코드를 유일하게 식별할 수 있는 다른 후보 키들

<br>

## 5. 슈퍼 키 (Super Key)

테이블의 각 행을 **유일하게 식별**할 수 있는 **하나 이상의 열(Column) 집합**

한 행을 고유하게 식별할 수 있으면 모두 슈퍼키가 될 수 있음.

- 슈퍼키는 기본키, 후보키, 대체키 등을 포함한 더 넓은 범위의 키
- 유일성은 만족하지만, 최소성은 만족하지 않을 수 있음
- e.g. 학생 테이블 
학번 - 주민번호 - 이름
<br>
학번 : 슈퍼키
<br>
주민번호: 슈퍼키
<br>
학번, 이름: 슈퍼키
<br>
주민번호, 이름, 학번: 슈퍼키
